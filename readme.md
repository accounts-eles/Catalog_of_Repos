# Dynamic GitHub Repository Catalog

A single-file HTML application that fetches and displays a GitHub user's public repositories via the GitHub REST API. It provides live search, topic filtering, card selection, bulk file downloading, and a print-to-PDF catalog view — all with no build tools or dependencies beyond a CDN-loaded Tailwind CSS.

---

## Features

- **Live repo fetching** — pulls up to 100 public repositories on page load using the GitHub REST API
- **User profile display** — shows avatar, name, bio, location, and website from the GitHub profile
- **Search** — filters cards in real time by repository name or description
- **Topic filtering** — dynamically generated pill buttons for each GitHub topic tag; click to filter by category
- **Repository cards** — each card shows a local preview thumbnail (with a placeholder fallback), description, language badge, topic tags, and last-updated timestamp
- **Selection & bundle download** — checkbox on each card; a sticky action bar tracks selected count and triggers bulk download of `index.html` and `README.md` from each selected repo
- **Select All / Clear All** — toggle all currently visible cards at once
- **Print / PDF export** — renders a clean two-column print layout, hiding UI chrome and showing only the catalog content

## Configuration

Two constants at the top of the `<script>` block control the app's behaviour:

| Constant | Default | Purpose |
|---|---|---|
| `HARDCODED_USERNAME` | `accounts-eles` | The GitHub username whose repos are fetched |
| `EXCLUDED_REPO_NAME` | `Catalog_of_Repos` | A single repo name to suppress from the listing |

Change these values to point the catalog at any GitHub account.

---

## Repository Card Requirements

Only repositories that meet **both** of these conditions are shown:

1. The repository name does not match `EXCLUDED_REPO_NAME`
2. The repository has **at least one GitHub topic tag**

Repositories without topic tags are silently excluded.

---

## Preview Thumbnails

Cards look for a local image at `./previews/<repo-name>.png`. If the file is not found, the card falls back to a placeholder image generated by [placehold.co](https://placehold.co) using the repo name as the label.

To add thumbnails, create a `previews/` folder alongside `index.html` and add a `.png` for each repository named exactly after the repo.

---

## Download Bundle

Selecting one or more cards and clicking **Download Bundle** will attempt to fetch two files from each selected repository's default branch on GitHub:

- `index.html`
- `README.md`

Files are downloaded individually via the browser's native download mechanism, named `<repo-name>-index.html` and `<repo-name>-README.md`. Files that do not exist in the repo are silently skipped.

---

## Print / PDF Export

Clicking **Print Catalog** triggers `window.print()`. The stylesheet includes a `@media print` block that:

- Hides all interactive UI (header, search, filters, action bar, checkboxes)
- Switches the card grid to two columns
- Removes shadows and expands truncated descriptions
- Shows the raw project URL beneath each card's language badge

For best results, use **Save as PDF** from the browser's print dialog with background graphics enabled.

## Tech Stack

| Layer | Technology |
|---|---|
| Markup | HTML5 |
| Styling | [Tailwind CSS](https://tailwindcss.com/) via CDN |
| Fonts | Inter (system fallback) |
| Logic | Vanilla JavaScript (ES2017+) |
| Data | [GitHub REST API v3](https://docs.github.com/en/rest) |

No build step, no npm, no frameworks. The entire app is a single `index.html` file.

---

## Local Usage

1. Clone or download this repository.
2. Open `index.html` directly in a browser — no server required for basic browsing.
3. For the bundle download feature to work, the browser must be able to reach `raw.githubusercontent.com`. A local file open (`file://`) is sufficient; no local server is needed.
4. To customise the catalog, edit `HARDCODED_USERNAME` and `EXCLUDED_REPO_NAME` in the script block.

> **Note:** The GitHub API allows up to 60 unauthenticated requests per hour per IP address. The app makes 2 requests on load (user profile + repo list). For higher rate limits, you would need to add an authenticated request header, which is not implemented in this version.
